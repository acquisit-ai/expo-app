# 系统集成架构

*此文档定义实际的跨组件通信模式、数据流和集成策略，重点涵盖认证系统、状态机模式、provider层次架构，以及视频系统的三层FSD架构参考实现。*

## Feature-Sliced Design 三层集成架构

### 视频系统企业级参考实现

本项目的**视频系统**展示了完美的FSD三层架构集成模式，作为跨组件通信和状态协调的标杆实现：

#### 三层协调架构
```
entities/video/              # 数据层 - 企业级状态机模式
    ↓ 状态提供
features/video-player/       # 功能层 - 策略模式UI架构
    ↓ 组件组合
widgets/video-player-section/ # 组件层 - SharedValue动画协调
```

#### 跨层通信模式
- **Entity → Feature**: 通过hooks提供状态和操作，零prop drilling
- **Feature → Widget**: Context Provider模式，计算值缓存
- **Widget层协调**: SharedValue桥接，跨widget动画同步

#### 集成优势
- **状态机模式**: 三状态同步(Zustand ↔ SharedValues ↔ VideoPlayer)
- **策略模式UI**: 动态控件选择，适配不同显示模式
- **性能优化**: 选择性订阅，内存安全管理
- **企业级质量**: 完整的错误处理、日志记录、生命周期管理

## React Native 应用集成架构

### 系统边界定义
**核心系统**: React Native单词学习应用，基于Feature-Sliced Design (FSD)架构
**外部集成**: Supabase BaaS (认证、数据库)，Expo平台服务
**数据边界**: 本地SQLite数据库 + Supabase云端数据同步
**安全边界**: Supabase认证系统，行级安全(RLS)数据隔离

### Provider层次通信架构
- **ThemeProvider → GlassProvider**: 主题状态(isDark)传递给玻璃态效果系统
- **ThemeProvider → ToastProvider**: 主题样式集成到Toast通知系统
- **ToastProvider → AuthProvider**: 认证操作错误反馈通过Toast系统
- **AuthProvider → UI Components**: 认证状态和操作通过React Context传递
- **Supabase Integration**: 直接集成模式，无中间抽象层

## 认证数据流架构

### 认证状态流向
```
[用户操作] → [UI组件状态机] → [AuthProvider全局状态机] → [AuthOperations类]
                                          ↓
[Toast通知] ← [错误处理] ← [Supabase API] ← [API冷却保护]
```

### 状态一致性策略
- **状态机模式**: 认证流程使用严格的状态机，防止状态不一致
- **内存安全检查**: 所有异步操作包含组件挂载状态检查
- **批量状态更新**: 减少React重渲染周期，提升性能
- **派生状态计算**: 从核心状态计算UI状态，避免重复状态变量

### 缓存和持久化策略
- **会话缓存**: Supabase SDK自动处理会话持久化到AsyncStorage
- **主题偏好缓存**: 用户主题选择持久化到AsyncStorage
- **玻璃态样式缓存**: GlassStyleFactory实现双层缓存系统
- **Toast配置缓存**: 开发模式下的性能监控和缓存清理

## Supabase集成模式

### 认证API集成
**直接Supabase SDK调用**:
- 无中间抽象层，直接使用Supabase客户端
- 统一错误处理：`getFriendlyErrorMessage`函数转换用户友好错误
- API速率限制：内置冷却保护机制(sendCode: 60s, verify: 3s)
- 响应类型：利用Supabase SDK的TypeScript类型系统

**认证流程集成**:
- JWT会话管理：Supabase SDK自动处理令牌刷新
- onAuthStateChange监听器：实时同步认证状态变化
- 会话持久化：跨应用启动的会话恢复
- 多认证方式：邮箱+密码、OTP Magic Link统一集成

### 状态机事件处理
- **状态转换事件**: 认证状态机的INITIAL_SESSION, SIGNED_IN, SIGNED_OUT事件
- **异步操作队列**: 冷却保护确保操作按顺序执行，防止并发冲突
- **错误事件传播**: 认证错误自动传播到Toast系统和UI组件
- **内存安全事件**: 组件卸载事件立即停止所有异步状态更新

### 数据持久化集成
- **本地SQLite + Drizzle ORM**: 类型安全的本地数据存储
- **Supabase云端数据库**: PostgreSQL行级安全(RLS)数据隔离
- **离线优先架构**: 应用可在无网络环境下运行
- **增量数据同步**: 与Supabase云端服务的数据同步

## Toast通知系统集成

### 全局Toast通知集成
**单例模式ToastManager**:
- **统一通知入口**: `toast.show(config)`提供全局通知服务
- **主题集成**: 自动适配深色/浅色模式的BlurView效果
- **队列管理**: 支持最多3个同时显示的Toast，自动队列管理
- **类型安全**: 品牌类型和运行时验证

**认证错误集成模式**:
- **错误分类**: 自动区分用户错误和系统错误
- **友好错误消息**: `getFriendlyErrorMessage`转换技术错误为用户友好提示
- **冷却警告**: API冷却保护触发时自动显示剩余时间
- **操作反馈**: 认证操作成功/失败的实时通知

### 监控和可观测性
- **结构化日志**: 使用`src/shared/lib/logger.ts`统一日志记录
- **认证事件监控**: 登录尝试、认证失败、速率限制的安全日志
- **性能监控**: 开发模式下的缓存大小监控和警告
- **内存安全追踪**: 组件挂载状态跟踪和内存泄漏防护
